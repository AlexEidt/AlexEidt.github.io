Game of Life Simulator

Dec 2020 $\rightarrow$ Jan 2021

This is a simulator I built for Conway's Game of Life. The Game of Life is a simple one-player game (or simulation) where a set of four simple rules dictates which cells on a board survive into the next generation. The user gives a certain starting condition and watches how cells on a board grow and change.

The user can draw any starting condition they'd like and start/stop the simulation to make adjustments. They can also step through each generation of the simulation to more closely see the changes. The user can save the current state of the board to reload later as well as capture the current state as a transparent PNG or record the simulation as a transparent GIF.


The app is built in Kotlin and the UI is made with Kotlin Swing. I used Java for the PNG screenshot and GIF creation. Below are some examples of what the PNG screenshot and GIF recording look like.

UW Course Planner

Jul 2019 $\rightarrow$ Sept 2019

I got the idea for this project while at the AngelHack 2019 Hackathon in Seattle. I went with a friend and pitched the idea. It was my first time working with Python, so I was learning the language while working on this project. We worked deep into the night and unfortunately didn't get anywhere. The next day, we both left the Hackathon early, and I took it upon myself to go back to the drawing board and rebuild the project from the ground up. After putting most of my summer break into it, I finally completed the project and had built a foundation in Python.

After successful completion of the project, I reached out to UW IT to let them know about it. I presented the project to the UW IT Academic Experience Design & Delivery Team to improve MyPlan, the existing course management tool at UW. The meeting went well and actually led to a course keyword search feature being added to MyPlan!

The UW Course Planner is basically a course management/lookup tool for all three UW Campuses. It features prerequisite trees for every course that recursively outline all prerequisites for any given course.


It also features schedule visualization on a campus map so that students can see where their courses are located on campus.


Additionally, it features a keyword search for course descriptions.


In order to get all the data to display on this website, I built a library called uwtools that parses the UW Time Schedule and Course Catalogs into Dictionaries and DataFrames. It's basically a web scraper using BeautifulSoup and pandas.

The backend of this web app was made with Python using the Flask framework. The pandas library is used to manage course data and GraphViz is used to create the prerequisite trees. The frontend is made using HTML, CSS, Bootstrap and JavaScript . The map is made with Leaflet.
